HoardBoard Case Study

HoardBoard is a cloud storage concept for fans to save and share their favorite images and notes.

I created a mobile and desktop design through user research, branding research, and prototype testing.

  • User Research
  • Branding
  • Visual Design
  • Prototypes Testing
  • Information Architecture
View Prototype

    Problem

  • My project was to work with a client in designing a cloud storage platform that fit into the current overcrowded market. It needed functions that allowed sharing, collaboration, saving, uploading, and organization of files.
  • The project brief was very broad, so my task was to collect research and design a project that would be marketable in the current space.

    Deliverables

  • Clickable prototype (Figma)
  • High Fidelity Mockups
  • Style Guide

    Tools

  • Figma
  • Pen and paper
  • Usability Hub
  • Google Survey

Solution

  • Through research, I was able to clarify and hone the project scope, settling on creating a product built for fans of tv shows and other fictional content.
  • The solution utilized the following choices:
    • - use of search and filter to make organization easier
    • - visual choices more suitable for the target audience
    • - capability to do all the required tasks in the project brief

Phase 1. User Research

Goal: to understand users and their experiences with current cloud storage apps

User Survey

Goal: to find why users like their preferred cloud storage application, and how they used them

  • According to my survey of 25 people, the overwhelming majority of users preferred Google Drive.
  • Users most value:
    • - price
    • - ease of use
    • - ease of sharing
  • Most important features:
    • - sharing files
    • - uploading files
    • - collaboration

Connections & Trends

  • 1/2 of the users surveyed were hobbysists.
    1. 1. They wanted access to cloud storage anywhere.
    2. 2. They wanted to share files easily with others.
  • If a user wanted to share a file, they preferred using a link over email.

User Stories

Next, I created a list of user stories to give direction to the app flow, and to list out specific tasks a user would be able to complete

Priority Task
High As a returning user, I want to create and upload an image/folder so I can store my files
High As a returning user, I want to navigate and move my files or folders so I can organize my files.
High As a returning user, I want to share my files or folders so other people can see my files.
Medium As a new user, I want to sign in/sign out of my account.
Medium As a returning user, I want to edit, delete, or copy my file or folder so I can have full control over what’s in my collection.
Low As a returning user, I want to search for a file so I can locate and work with it.
Low As a returning user, I want to view my files as a list or as a grid so I can quickly locate the file I want to work with.

Phase 4. User Testing

Goal: to conduct preference and usability tests to improve visual design and user experience

Preference Tests

After applying the HoardBoard style to my wireframes, I tested my high fidelity mockups.

I utilized UsabilityHub to gather user preferences for design decisions, such as coloring, element styling, and background design.

Preference Test #1: Sidebar navigation color test

Which sidebar color combination do you prefer?

Winner!

Users preferred the gray sidebar because it did not distract them from the main content of the application.

In addition, users preferred it when the selected folder had a white background for better visibility.

Other versions:

Preference Test #2: Create button styles

Which upload/create button style do you prefer?

Winner!

I created two main designs - a menu that expanded vertically, and a menu that expanded in a circular manner.

The feedback I received was spread pretty evenly between the four options. People liked the straightforward style of the vertical menu, while others liked the fun vibe the circular mannar gave.

Other versions:

Preference Test #3: Empty dashboard onboarding screen

Which empty dashboard design do you prefer?

Initial test:

Users preferred seeing an image as an onboarding background, as compared to text only.

Final product:

I ultimately decided to use BOTH the image and text to convey the idea of an empty dashboard, as suggested by one of the users.

Usability Tests

Next, after completing the preference tests, I conducted three live user tests through skype or in-person meetings.

Once again, I asked my users to complete four tasks:

  1. 1. sign up for an account
  2. 2. create a note
  3. 3. share an image
  4. 4. toggle grid to list
Challenge #1: Placement of the grid/list toggle button

Grid/List toggle button confused people because it was in the navigation bar. I moved it closer to the meat of the content, where you interact with your files.

Toggle button in navigation

Moved toggle to main organization area

Challenge #2: Landing page design elements

Version 1

Backgrounds and pictures looked inconsistent and clased with the HoardBoard style.

Version 2

Some section headings looked like buttons, which caused confusion on what was clickable.

Final

I removed the inconsistent styles and images, and increased the space between elements.

Challenge #3: Create button's hover capability

It wouldn’t be good to have a hover menu immediately pop up and cover the button because some users may accidentally click on a menu item when they were expecting to click on the button.

Conclusion

    Through this project, I learned the importance of studying the user research, and actually following through with the findings.

    I have a better understanding of how information architecture and visual design can address user pain points. They must all come from designing with the user in mind.

    I found my love for usability testing by talking to users and coming to understand and empathize with their problems and points of view.

Reflection

    Some of the biggest challenges I had was taking constructive criticism and determining what kind of mood I wanted to go with for the final visual design and brand.

    There were many assumptions I made that ultimately kept my project from reaching its fullest potential.

    This reinforced the face that it was necessary for me to rely on my research and follow the process based on that research.

Future

    I plan on spending more time in the beginning, user research stage of the work.

    It is a great idea to plan ahead and be prepared to create design based on research results.

    If I had planned more in the future, I would not have had to spend so much time scratching my head over bad design choices.
